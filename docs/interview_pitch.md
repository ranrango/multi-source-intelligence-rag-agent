# 面试讲述稿

## 1 分钟版本

我做了一个多源情报 RAG 分析 Agent。它不是简单把资料丢给大模型问答，而是先从新闻、社媒、传感器日志、内部通知和分析员简报中检索证据，再调用实体抽取、时间线整理和风险评估工具，最后生成带引用来源和不确定性说明的研判报告。

这个项目重点展示 Agent 工程能力：RAG 负责找证据，Tool Calling 负责结构化分析，Trace 负责审计，报告生成负责交付给业务方。

## 面试追问

### 为什么不是普通 RAG？

普通 RAG 更像“检索后回答”，但安全态势分析需要多步骤判断：证据归并、实体关联、时间线复盘、风险评分和不确定性说明。Agent 的价值是把这些步骤编排起来，并让每一步可控、可追踪。

### 如何减少幻觉？

我会限制回答必须基于 evidence，输出引用编号；当证据不足时要明确说不确定。工程上会加证据充足性判断、冲突证据检测、工具白名单和 trace 审计。

### 上线后怎么评估？

至少评估四类指标：召回是否命中关键证据、回答是否引用正确来源、风险等级是否符合人工标注、端到端延迟和失败率是否稳定。
